Build a "Ghost Player Correction Animation" module for a tennis biomechanics analysis application.

CONTEXT The application already analyzes tennis videos using OpenCV and MediaPipe Pose. The system already calculates 25 biomechanics metrics and stores the 33 MediaPipe pose joints for every frame.
This module should NOT perform pose detection again. It should only use the stored skeleton data and metrics.

GOAL
Generate an animation that shows the player's skeleton compared with an "optimal correction skeleton" focusing ONLY on the highest priority mistake.
Visualization concept:
Blue Skeleton = Player Green Skeleton = Corrected motion
Both animations play together so the player can visually see how to correct the movement.

STEP 1 – DETERMINE PRIORITY CORRECTION
Use the metrics and optimal ranges to determine the biggest deviation.
Example:
player_metrics = { shoulder_rotation: 48, hip_rotation: 62, knee_bend: 40 }
optimal_ranges = { shoulder_rotation: [70,100], hip_rotation: [60,80], knee_bend: [35,55] }
Compute deviation score:
deviation = distance from optimal range
Select the metric with the largest deviation.
Example output:
priority_correction = "shoulder_rotation"
All the metrics are stored in metrics table metric_values column

STEP 2 – GENERATE CORRECTION SKELETON
Using the player's stored skeleton frames:
Adjust only the joints related to the priority correction.
Examples:
shoulder_rotation correction:
* rotate shoulder line
* adjust elbow and wrist path
hip_rotation correction:
* rotate hip axis
* adjust torso orientation
knee_bend correction:
* modify knee angle
Skeleton frames stored in metrics table ai_diagnostics column
Do NOT modify unrelated joints.

STEP 3 – RENDER GHOST ANIMATION
Render two skeleton animations simultaneously:
Player skeleton (blue) Optimal correction skeleton (green)
Use MediaPipe joint topology.
Skeleton connections include: shoulders, elbows, wrists, hips, knees, ankles.
Use Canvas or Three.js for rendering.

STEP 4 – HIGHLIGHT ERROR
Highlight the incorrect joint in red.
Example:
shoulder_rotation error → highlight shoulder joints
Add a floating label:
"Rotate shoulders earlier"

STEP 5 – ANIMATION CONTROLS
Provide UI controls:
play pause frame scrubber slow motion: 0.25x, 0.5x, 1x

STEP 6 – VISUAL CORRECTION GUIDES
Add visual arrows showing the correction direction.
Example:
shoulder rotation → curved arrow around shoulder axis
knee bend → arrow showing deeper bend

STEP 7 – UI LAYOUT
Top panel:
Ghost skeleton animation
Bottom panel:
Correction insight
Example:
Priority Correction: Shoulder Rotation
Player Value: 48° Optimal Range: 70°–100°
Recommendation: Start shoulder rotation earlier during the unit turn.

PERFORMANCE REQUIREMENTS
Animation must run at 60fps. Support swings with up to 500 frames.
Do not reprocess video or run MediaPipe again.

OUTPUT
Create the following components:
GhostSwingAnimation SkeletonRenderer CorrectionDetector CorrectionVisualizer PlaybackControls
Organize the module so it can plug into the existing Performance metrics screen
